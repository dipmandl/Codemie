# Feature: Display Team Name on release cards when provided

## Story Information
- Story ID: CS-51
- Jira URL: https://mandlikdipak52.atlassian.net/browse/CS-51
- Priority: Medium
- Issue Type: Story

## Feature Overview
The form collects and stores `teamName`, but release cards do not display it. Add a conditional Team Name row/label to the card template and rendering logic.

## Business Goal
Improve ownership visibility and traceability for each release note by showing the responsible team when provided.

## Scope
**In scope**
- Add team name display to release cards.
- Hide the team row when teamName is empty.

**Out of scope**
- New filtering by teamName (separate story)
- Changing data model (already includes teamName)

## Functional Requirements
- If `teamName` is non-empty, show “Team: <teamName>” on the release card.
- If `teamName` is empty/blank, do not show the Team label/row.

## Non Functional Requirements
- No external dependencies.
- Keep layout clean and consistent.

## Existing Implementation
- `product_release_dashboard/index.html`
  - Form has Team Name input: `#teamName`.
  - Card template includes product/version, badge, title, description, date, edit button.
- `product_release_dashboard/script.js`
  - `readFormValues()` reads and persists `teamName`.
  - `setFormValues()` populates `teamName` during edit.
  - `renderReleaseList()` does not reference `teamName`.

## Proposed Changes
### 1) Update card template (`index.html`)
Add a dedicated element for teamName, e.g. between description and date:
```html
<p class="release-team" hidden></p>
```
(or a `<p>`/`<div>` with a nested span for value).

### 2) Update rendering (`script.js`)
In `renderReleaseList()`:
- Read `release.teamName`.
- If present (after trim):
  - set textContent to `Team: ${release.teamName}`
  - remove `hidden` (or toggle a `hidden` class)
- Else:
  - ensure the element stays hidden and has no text.

### 3) Style the team row (`styles.css`)
- Add `.release-team { margin: 0; color: var(--muted); }`
- Ensure spacing matches other metadata rows.

## File Changes

## Data Flow
- No change to persistence.

## UI Changes
- Release card shows Team row when present.
- Team row is omitted (hidden) when blank.

## API Changes
- None

## Database Changes
- None

## Validation
- Team name is optional; no new validation.

## Error Handling
- If `teamName` is undefined/null, treat as empty.

## Edge Cases
- Existing releases in localStorage that do not have `teamName` should render normally (row hidden).
- Whitespace-only teamName should be treated as empty (use `.trim()` in rendering condition).

## Security Considerations
- Render teamName via `textContent` to avoid XSS.

## Testing Plan
Manual:
1. Create release with Team Name set → card shows “Team: …”.
2. Create release with Team Name blank → no Team row.
3. Edit an existing release, add a Team Name → card updates to show Team.
4. Edit a release with Team Name, remove it → Team row hides.
5. Seed data (no teamName) renders without layout issues.

## Risks
- Minor layout changes; mitigate via consistent CSS and using `hidden` attribute.
